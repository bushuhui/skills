# 论文创新点挖掘 — 5-Skill 工作流

采用 5 步流水线：范式爆破 → 理论碰撞 + 异常值 → 方法论升级 → 贡献声明

## 核心架构

5 个专精的 AI Skill 串联工作，对既定主题进行多维度严苛审查，系统性地提炼并确立具有深度的学术创新点。

```
┌─────────────┐     ┌──────────────┐     ┌──────────────┐
│ 范式爆破手  │────>│ 理论碰撞机   │────>│ 异常值猎犬   │
│ (范式映射)  │     │ (跨学科碰撞) │     │ (边界条件)   │
└─────────────┘     └──────────────┘     └──────────────┘
                                                 │
                                                 ▼
                    ┌──────────────┐     ┌──────────────┐
                    │ 贡献声明     │<────│ 方法论降维   │
                    │ (价值升华)   │     │ (方法升级)   │
                    └──────────────┘     └──────────────┘
```

## 关键约束

1. **分步处理**：严格按 5 步流水线执行，严禁跨步跳跃
2. **原意保留**：所有创新建议必须基于用户提供的主题和文献，严禁凭空捏造
3. **显式靶标**：每一步输出必须显式界定"核心靶标"或"创新缺口"
4. **反增量式**：封杀所有"增量式"建议（如"在 X 国验证"、"加入一个新变量"），聚焦范式转移级创新
5. **跨学科严谨**：跨学科理论必须具有逻辑同构性，禁止生搬硬套或仅作隐喻

## 5 步流水线

### Step 1：构建研究共识（范式爆破手）

- 接收用户提供的**宽泛研究主题**和 **5 篇奠基性文献**
- 绘制"基准范式"（领域内所有人都默认正确的常识）
- 解剖底层假设，暴露理论软肋
- 输出：《范式拆解报告》，高亮极易被颠覆的理论盲区
- 详见 → [[references/skill-01-paradigm-breaker.md]]

### Step 2：突破传统框架（理论碰撞机 + 异常值猎犬）

并行执行两个 Skill：

- **理论碰撞机**：引入 3 个跨学科理论透镜，裂变混合型研究问题
- **异常值猎犬**：猎杀主流理论失效的边界条件
- 输出：《创新视角清单》，筛选最具颠覆性的方向
- 详见 → [[references/skill-02-theory-collider.md]]
- 详见 → [[references/skill-03-outlier-hound.md]]

### Step 3：优化方法论（方法论降维打击）

- 识别领域"标准/老套"方法论
- 提出不对称的方法论升级方案
- 解释为什么新方法能绕过旧方法的先天缺陷
- 输出：《方法论升级蓝图》
- 详见 → [[references/skill-04-methodology-strike.md]]

### Step 4：提炼贡献声明（价值升华官）

- 接收步骤 1-3 的输出结果
- 起草引言末尾的"本文贡献"段落
- 输出：3 点核心贡献声明，提前粉碎"增量式研究"攻击
- 详见 → [[references/skill-05-value-sublimate.md]]

## 用户交互模板

当用户触发此 Skill 时，按以下流程引导：

```
请提供以下信息：
1. 【研究主题】：你想探索的宽泛领域
2. 【核心学科】：你的主学科（如计算机科学、管理学等）
3. 【奠基文献】：5 篇该领域最具代表性的论文（标题或摘要）
4. （可选）当前遇到的创新瓶颈或审稿人批评

收到后，我将依次执行：
步骤一 → 范式拆解 → 步骤二 → 创新视角 → 步骤三 → 方法升级 → 步骤四 → 贡献声明
```
