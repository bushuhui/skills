# Analysis Dimensions

When analyzing a UI sketch/design image, cover ALL 6 dimensions for each screen.
Even if a dimension seems irrelevant, note "N/A" or "none visible" — never skip.

## 1. Layout Structure

- Overall layout pattern (e.g., sidebar + main, top-nav + content, card grid, split panel)
- Number and arrangement of major regions/sections
- Responsive hints (does it look mobile, tablet, desktop?)
- Spacing and alignment patterns (centered, left-aligned, etc.)

## 2. Components & UI Elements

For each distinct component/element visible:
- **Type** (button, input, dropdown, card, badge, avatar, checkbox, radio, toggle, etc.)
- **Label/Text** (what text does it display?)
- **Position** (which section/region is it in?)
- **State** (default, disabled, active, selected, error, etc.)
- **Variant** (primary, secondary, outline, ghost, etc. if apparent)

Common component types to inventory:
- Navigation elements (tabs, breadcrumbs, menus)
- Form controls (inputs, selects, checkboxes, radios, toggles)
- Data display (cards, tables, lists, badges, avatars)
- Feedback (alerts, toasts, progress bars, skeletons)
- Actions (buttons, links, icon buttons)

## 3. Interactions

- Click/tap targets and their expected behavior
- Form submission flows
- Navigation paths (what happens when you tap each link/button)
- Modal/drawer/sheet triggers
- Swipe/gesture interactions (if any)
- Loading states and transitions
- Error states and recovery paths

## 4. Visual Design

- Color usage (primary, secondary, accent, semantic colors)
- Typography hierarchy (heading sizes, body text, captions)
- Icon usage (navigation icons, status indicators, decorative)
- Spacing rhythm (padding, margins, gaps)
- Border/radius style (rounded, sharp, pill)
- Shadow/elevation levels

## 5. Data & State

- What data is displayed (static vs. dynamic)
- Loading states (skeletons, spinners, progressive loading)
- Empty states (no data, no results)
- Error states (network errors, validation errors)
- User-specific content (personalized vs. generic)

## 6. Accessibility

- Touch target sizes (minimum 44x44px recommended)
- Color contrast (text readability)
- Focus order and keyboard navigation hints
- Screen reader considerations (meaningful labels)
