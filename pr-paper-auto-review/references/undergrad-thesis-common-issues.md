# 本科毕业论文常见问题模式

## 基于 pi-llm-server OCR 转写后的典型问题

### 1. 末尾章节 OCR 错误
论文经 MinerU OCR 转 Markdown 后，**越靠后的章节越容易出现文字混乱**。第六章"总结与展望"尤其严重，可能整段文字变成乱码或碎片化。审稿时必须逐字校对末章。

### 2. 参考文献引用不匹配
论文正文中引用的文献编号与实际参考文献列表内容不一致（如正文 [2] 说的是 A，但参考文献 [2] 实际是 B）。常见原因是作者增删文献后未重新编号。

### 3. 公式编号缺失
Markdown 中的 `$$...$$` 公式往往缺少编号，毕业论文要求关键公式必须编号。

### 4. 图表标题位置
- 图标题应在图下方，表标题应在表上方
- OCR 可能将标题识别为普通段落

### 5. 列表项序号断裂
OCR 可能漏掉列表项的编号（如"4."缺失直接跳到"5."），需手动检查。

## 审稿意见模板结构
1. 总体评价（优点 + 不足）
2. 逐章节问题（按章节顺序，每章编号列出）
3. 写作与格式问题
4. 优先级汇总表（紧急/高/中/低）
5. 总结与答辩建议
